# 🏄 Surfer Search

놀란봇노예가 개발하는 효율적이고 강력한 개인형 검색엔진 프로젝트입니다.

## 1. 프로젝트 개요 (Overview)
**Surfer Search**는 웹의 파도를 타며 필요한 정보를 정확하게 낚아채는 검색엔진입니다. 방대한 웹 데이터를 단순히 수집하는 것에 그치지 않고, 사용자에게 가장 가치 있는 정보를 우선적으로 제공하는 것을 목표로 합니다.

## 2. 개발 방식 (How it works)
검색엔진은 크게 4단계의 파이프라인으로 구성됩니다:
1.  **Crawler (수집기)**: Scrapy 및 BeautifulSoup을 활용하여 타겟 웹사이트의 데이터를 효율적으로 수집합니다.
2.  **Parser (해석기)**: 수집된 HTML에서 불필요한 태그를 제거하고, 핵심 텍스트와 메타데이터(Title, Description 등)를 추출합니다.
3.  **Indexer (색인기)**: 추출된 데이터를 검색 가능한 형태로 저장합니다. 초기에는 가벼운 **SQLite**나 **Inverted Index** 구조를 사용하며, 확장 시 **Meilisearch** 도입을 검토합니다.
4.  **Search & Ranking (검색 및 순위)**: 사용자의 질의어(Query)에 대해 TF-IDF 또는 BM25 알고리즘을 사용하여 관련성이 높은 순서대로 결과를 정렬합니다.

## 3. 타 검색엔진 대비 강점 (Advantages)
-   **초경량화**: 불필요한 광고나 추적 스크립트 없이 오직 검색 결과에만 집중합니다.
-   **개인 맞춤형 인덱싱**: 사용자가 관심 있는 특정 도메인이나 커뮤니티를 집중적으로 크롤링하여 구글보다 더 깊은 정보를 제공할 수 있습니다.
-   **투명한 랭킹**: 어떤 기준으로 검색 결과가 상단에 노출되는지 사용자가 로직을 직접 확인할 수 있습니다.

### [Week 1: Core Engine Development]
-   **Day 1-2**: 환경 설정 및 기본 Crawler 구현 (특정 도메인 대상)
-   **Day 3**: Parser 개발 - 본문 추출 로직 최적화
-   **Day 4**: Indexer 구축 - 데이터베이스 스키마 설계 및 저장 로직 구현
-   **Day 5**: 기초 검색 로직 (Keyword Match) 구현 및 테스트

### [Week 2: Advanced Features & UI]
-   **Day 6-7**: Ranking 알고리즘 개선 (BM25 도입 및 가중치 조절)
-   **Day 8-9**: Web UI 개발 (Next.js/Tailwind CSS 기반) - 깔끔하고 빠른 검색창
-   **Day 10**: 검색어 추천 및 검색 히스토리 기능 추가
-   **Day 11**: 배포 환경 설정 및 성능 최적화 (캐싱 처리)
-   **Day 12**: 전체 시스템 통합 테스트 및 버그 수정
-   **Day 13-14**: 최종 문서화 및 프로젝트 완료 보고
